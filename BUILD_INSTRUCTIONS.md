# Build & Packaging Guide

This guide details how to compile the KlosedLoop application for different platforms. The project uses [Wails](https://wails.io), which compiles the Go backend and bundles the React frontend into a single native binary.

## Prerequisites

Before building, ensure your environment is set up correctly.

### Common Requirements
- **Go**: v1.22 or later
- **Node.js**: v18 or later
- **Wails CLI**: Install via `go install github.com/wailsapp/wails/v2/cmd/wails@latest`

> **First-time setup**: `frontend/wailsjs/` is auto-generated by Wails and not committed to the repository. Run `wails dev` once after cloning to generate the Go-to-TypeScript bindings before attempting a production build.

---

## 1. Building for macOS

To create a `.app` bundle for macOS:

### Steps
1.  **Generate Icons**:
    -   Ensure `build/appicon.png` exists (1024x1024 recommended).
    -   Wails automatically generates the `.icns` file during the build.

2.  **Build Command**:
    ```bash
    wails build
    ```
    -   **Output**: `build/bin/klosedloop.app`

3.  **Universal / Specific Architectures**:
    -   To ship for Apple Silicon specifically:
        ```bash
        wails build -platform darwin/arm64
        ```
    -   To ship a universal binary (if manually configured):
        ```bash
        wails build -platform darwin/arm64,darwin/amd64
        ```

4.  **Signing & Notarization**:
    -   **Crucial**: macOS blocks unsigned apps by default.
    -   You must use `gon` or `codesign` to sign the generated `.app` bundle with your Apple Developer Certificate.

---

## 2. Building for Windows

**Note**: Building for Windows is best done **on Windows**. Cross-compilation from macOS is difficult due to CGO and `WebView2` dependencies.

### Steps (On Windows)
1.  **Install NSIS**: This is required to generate the installer (`setup.exe`).
2.  **Build**:
    ```powershell
    wails build -nsis
    ```
    -   **Output**: `build/bin/klosedloop.exe` and an Installer file.

### Cross-Compilation Strategy
If you are on macOS and need a Windows build, it is highly recommended to use **GitHub Actions** or a Windows VM rather than attempting to cross-compile locally.

---

## 3. Building for Linux

**Note**: Must be built on a Linux environment (e.g., Ubuntu).

### Steps (On Linux)
1.  **Install Dependencies**:
    ```bash
    sudo apt install libgtk-3-dev libwebkit2gtk-4.0-dev
    ```
2.  **Build**:
    ```bash
    wails build
    ```
    -   **Output**: `build/bin/klosedloop`

---

## 4. Automating with GitHub Actions

We recommend using CI/CD to generate binaries automatically. Create a `.github/workflows/release.yml`:

```yaml
name: Build and Release
on:
    push:
        tags:
            - "v*"

jobs:
    build:
        strategy:
            matrix:
                os: [ubuntu-latest, windows-latest, macos-latest]
        runs-on: ${{ matrix.os }}
        steps:
            - uses: actions/checkout@v3
            - uses: duncanba/action-wails-build-linux@v1
                with:
                    build-name: klosedloop
            # ... (Add steps to upload artifacts to GitHub Release)
```
